- Handler interfaces (IDeathHandler, IActorEnterHandler, etc.): use `any` for forward-declared types to avoid circular deps

### Design Decisions
- GameModelRef: avoids circular imports — Entity/Floor/Tile read GameModelRef.main instead of importing GameModel directly
- Entity stores inlined into Floor.ts — StaticEntityGrid (2D array) for tiles/grasses/items/triggers, MovingEntityList (list + lazy grid cache) for bodies
- Entity type discrimination: uses duck-typing ('hp' in e → body, '_isGrass' in e → grass, etc.) since TS doesn't have C# `is` operator for our inheritance chain. Will refine when Body/Actor/Grass classes exist.
- PathfindingManager inlined as Floor.findPath() — A* with adjacentTiles (8-dir Moore neighborhood), weighted by tile/body collision
- FloorEnumeratorExtensions inlined as Floor methods — enumerateLine (Bresenham), enumerateCircle, BFS, etc.
- No uuid dependency — Entity guids use incrementing counter (`e-0`, `e-1`, ...) since we don't need persistence

### Design Decisions
